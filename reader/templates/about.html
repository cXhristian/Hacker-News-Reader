Hacker News Reader

A small project aiming to provide a better user interface for Hacker News.

Name is subject to change due to other (web)apps using the same name.

This small web-app is first of all a project I started as a tool for learning Python and Django. I am very new to both and have earlier used PHP for web programming.

The biggest reason I started this project was because I wanted to browse HN on Android phone and the web page was hard to read. And all the apps I could find on Play Store was lacking some functions that made it hard to use.

I am using Bootstrap as a CSS framework (with a still very unfinished theme), Sass for CSS, BeautifulSoup for HTML parsing, Django as a web framework and PostgreSQL as the database.
And for hosting I am using AppFog at the moment on their free plan.
You can check requirements.txt on Github if you are actually interested in what modules I use.

The site is running on a 20min cache for the front page (one cache for each sorting) and 20min cache for each story. The plan is to run a worker for fetching the front page (and just the default front page) every hour or so to provide a better coverage of stories.

Currently most of the site does not use ajax or similar techniques for sorting etc, but that is planned as soon as I get the site to work fine without javascript. I don't want to site to end up being an unnavigable mess for noscript users.

I am also planning to make an Android app with a similar function as this theme, but I have no prior experience with Android programming and limited knowledge of Java so it might take a while.

The project is available open-sourced at Github. Feel free to look over the code and give me tips as I am sure as a novice I am doing many mistakes. :-)
